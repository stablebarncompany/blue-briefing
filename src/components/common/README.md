# Common Components

Shared, reusable UI building blocks used across multiple feature areas (buttons, inputs, badges, empty states, and similar primitives).

Keep components small, typed, and theme-driven. Do not hardcode colors; import values from `src/theme`.

## Components

- `AppText` — themed text
- `AppButton` — primary / secondary / ghost actions
- `AppCard` — bordered surface panels
- `IconButton` — compact icon press target
- `SecurityStatus` — green “Agency Secure” indicator
- `SectionLabel` — uppercase section eyebrow
- `EmptyState` — spacious placeholder card
